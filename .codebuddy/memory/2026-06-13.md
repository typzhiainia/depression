# 2026-06-13 工作记录

## 安全审计与修复

### 发现并修复的安全漏洞：

**严重 (High)**
1. **CSRF 保护完全缺失** — 所有表单和 API 接口无 token 验证
   - 修复：在 config.php 添加 `csrf_token()` / `csrf_field()` / `csrf_verify()` 三个函数
   - api/public.php 和 api/admin.php 添加 POST 请求 CSRF 校验
   - test.php 的 fetch 请求添加 X-CSRF-Token header + body token
   - admin/index.php 的所有 AJAX 请求（loadRecords/deleteRecord/submitPasswordChange）添加 CSRF token

2. **数据库文件可公开下载** — 无 .htaccess 保护，data/*.db 可直接通过 URL 访问
   - 修复：创建 .htaccess 文件，阻止 .db/.sqlite/.sql 等敏感文件访问
   - 添加安全响应头：X-Frame-Options, X-Content-Type-Options, XSS-Protection, CSP, Referrer-Policy

**中等 (Medium)**
3. **$levelColor CSS 注入** — result.php 未验证直接输出到 HTML 属性/样式
   - 修复：添加正则校验 `/^#[0-9a-fA-F]{6}$/`

4. **$assessmentType XSS 风险** — test.php 未转义嵌入 JS
   - 修复：改用 `json_encode()` + `htmlspecialchars()`

5. **LIKE SQL 注入** — admin.php 搜索参数未转义通配符 `%` `_`
   - 修复：使用 `addcslashes($search, '%_\\')` 转义

6. **SQL LIMIT 注入** — public.php 第146行 LIMIT 直接拼接字符串
   - 修复：改为参数化查询

7. **管理员暴力破解无保护** — 登录接口无速率限制
   - 修复：基于 session 的 5次失败/15分钟锁定机制，含倒计时 UI

**低 (Low)**
8. **Session 安全配置不足** — 缺少 cookie_secure, samesite, gc_maxlifetime
   - 修复：补充完整 Session 安全配置

### 修改的文件列表：
- config.php (CSRF Token 系统 + Session 安全配置)
- .htaccess (新建，安全防护)
- api/public.php (CSRF 校验 + SQL 注入修复)
- api/admin.php (CSRF 校验 + LIKE 转义)
- test.php (CSRF token + htmlspecialchars + json_encode)
- result.php ($levelColor 正则校验)
- admin/index.php (CSRF_TOKEN 常量 + AJAX 请求带 token)
- admin/login.php (防暴力破解 + 锁定倒计时)
